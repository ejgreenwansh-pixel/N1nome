Privileged Container Escape via Kernel core_pattern Usermode-Helper

Linux 内核有一个机制：当进程崩溃（收到 SIGSEGV 等信号）时，内核会读取 /proc/sys/kernel/core_pattern 的内容来决定如何处理 core dump。

如果 core_pattern 以 | 开头，内核会把管道后面的内容当作一个 用户态程序，并以 root 权限 在 主机 PID namespace 中执行它。

也就是说： echo ”|/path/to/host/upperdir/exploit.sh” > /proc/sys/kernel/core_pattern

之后任何进程崩溃，内核都会以 host root 身份执行 /path/to/host/upperdir/exploit.sh。
