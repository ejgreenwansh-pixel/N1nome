CVE-2026-30225 & Cmd Injection

上面那个 advisory 影响版本 < 3000.11.1，我们正好是 3000.10.0。

简单说就是 RestartAction 这个 API 在内部重新构建请求时把原始调用者的 cookie / auth header 丢了，认证回退到 guest。如果一个受限用户（比如 exec: false）拿到了 executionTrackingId，就可以通过 RestartAction 以 guest 身份绕过去执行。

CVSS 5.3，利用条件：

  1. guest.exec = true ✅
  2. 受限用户 exec = false
  3. RestartAction 可访问 ✅

这台机器虽然没有配置受限用户，但这个洞存在；而且对我们来说更简单——我们本身就是 guest，直接调 API 就行，不需要绕。

OliveTin 的 REST API 有两个端点：

AI总结

完整攻击链：

  1. NFS 挂载 → PDF 泄露 Roundcube 凭据
  2. Roundcube 登录 → sarah 邮件泄露 kevin:Tr0ub4dor&3
  3. OpenSTAManager (support_001) → CVE-2025-69212 RCE → www-data shell
  4. Roundcube 数据库 → OpenSTAManager 数据库 (brollin:Fri3nds@9099)
  5. zz_users 表 → haris 的 bcrypt hash → 破解 → haris:K3!b3r0s
  6. haris SSH 登录
  7. OliveTin 命令注入 → backup_database 的 db_pass 参数闭合单引号 → chmod u+s /bin/bash
  8. /bin/bash -p → root

OliveTin 这里其实两个问题叠加：

  • 命令注入：参数值未经 shell 转义直接拼进命令字符串，password 类型不限制特殊字符
  • CVE-2026-30225：RestartAction 认证上下文丢失，受限用户可绕过 ACL 以 guest 执行。本机虽然无受限用户，但如果 guest.exec=false 且存在受限用户，这个洞就很有价值

修复：升级 OliveTin ≥ 3000.11.1，对参数做 shell 转义，defaultPermissions.exec: false，authRequireGuestsToLogin: true。
