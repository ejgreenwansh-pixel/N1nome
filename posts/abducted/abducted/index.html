CVE-2026-4480

google 搜索可以找到HTB发布过一个blog——War Room: CVE-2026-4480, when a Samba print job name is a shell command

CVE-2026-4480 是一个高危的 Samba 打印子系统命令注入漏洞（OS Command Injection），允许未经认证的远程攻击者实现远程代码执行（RCE）。CVSS 分数通常评为 9.8（Critical）。

Samba 在处理打印作业时，支持通过 smb.conf 中的 print command 参数来自定义打印命令。例如： print command = /usr/bin/lpr -P%p -J%J %s 其中 %J 是一个替换变量（substitution），它会被替换成客户端提交的打印作业描述（Job Description）。 问题在于：Samba 在替换 %J 时，没有对作业描述字符串进行 shell 元字符转义（如 ; & | $ ( ) { }` 等）。

攻击者可以控制 Job Description 字段。 Samba 直接把这个字符串拼接到 print command 中，并通过 shell 执行。 结果：攻击者可以注入任意 shell 命令。
